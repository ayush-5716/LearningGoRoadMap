1. Go is a compiled language like rust.
2. We can use go build "classname.go" to build an exe which. 
This will convert the go code to binary code and produce executable exe which can run on any device, even if go is not installed there.
In interpreted language , at runtime this conversion is done. Python is interpreted. 
3. Garbage Collection 
In java , there is automatic GC , which means , the JVM handles the garbage collection, JVM in itself is heavy on ram usage 
In Rust or C++ , this is done manually by the user, to be handled , and it is faster and light on ram
Go lies in between both, as the Garbage Collection is automatic but also there is no VM, the garbage colleciton is logic is present inside every compiled Go codeS

9. string formatting -

fmt.Println - simply prints as string
fmt.Printf - used for formated Printing
fmt.Sprintf() - used to return a formatted string (for ex. to store it in a variable)

const name = "Sual Goodman"
const age = 30.5

msg := fmt.Sprintf("Hi %s , your age is %f", name, age)

fmt.Println("this is the message : ", msg)

10. Printing float upto certain decimal points 

msg := fmt.Sprintf("Hi %s , your age is %.1f", name, age)

13. returning multiple values from a functions

func operations(x int, y int) (int, int, int) {
	return x * y, x - y, x + y
}

To reference all the values usage - 

mul, sub, sum := operations(10, 5)

To ignore some and refrence some, for example to ignore sub in this case - 

 mul , _, sum := operations(10, 5)
